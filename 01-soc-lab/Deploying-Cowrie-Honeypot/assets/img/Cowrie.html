Tutorial Install Cowrie Honeypot pada Ubuntu 20.04 Menggunakan Docker

Feren Public
3 min read
·
Nov 1, 2025

--

Cowrie adalah honeypot SSH dan Telnet yang dirancang untuk mencatat serangan brute force dan aktivitas malicious lainnya. Dalam tutorial ini, kita akan menginstal Cowrie menggunakan Docker pada Ubuntu 20.04.

Prasyarat

  • Ubuntu 20.04 LTS
  • Docker terinstall (ikuti tutorial install Docker sebelumnya)
  • Akses sudo/root
  • Koneksi internet

Langkah 1: Ubah Port SSH Default

Pertama, kita perlu mengubah port SSH default untuk menghindari konflik dengan Cowrie.

Edit Konfigurasi SSH:

sudo nano /etc/ssh/sshd_config

Cari baris:

#Port 22

Ubah menjadi:

Port 22222

Restart SSH Service:

sudo systemctl restart ssh
sudo systemctl status ssh

Verifikasi koneksi SSH baru:

ssh username@localhost -p 22222

Langkah 2: Install Docker (Jika Belum)

Jika Docker belum terinstall, ikuti tutorial install Docker pada link berikut:
Tutorial Install Docker pada Ubuntu 20.04

Verifikasi Docker terinstall:

sudo docker --version

Langkah 3: Download Cowrie Docker Image

Cari Cowrie Image di Docker Hub:

sudo docker search cowrie

Download Cowrie Image:

sudo docker pull cowrie/cowrie

Verifikasi Image Terdownload:

sudo docker images

Langkah 4: Menjalankan Cowrie Container

Jalankan Cowrie Container:

sudo docker run -d \
  --name cowrie \
  -p 22:2222 \
  cowrie/cowrie:latest

Penjelasan parameter:

  • -d: Run container dalam mode detached (background)
  • --name cowrie: Beri nama container "cowrie"
  • -p 22:2222: Map port 22 host ke port 2222 container
  • cowrie/cowrie:latest: Image yang digunakan

Langkah 5: Monitoring dan Testing

Cek Status Container:

sudo docker ps

Monitor Log Real-time:

sudo docker logs -f cowrie

Testing Cowrie Honeypot:

Dari host atau mesin lain, coba SSH ke Cowrie:

ssh root@<ip_address_cowrie>

Password yang bisa dicoba:

  • password
  • admin
  • 123456
  • root

Command Management yang Berguna

Melihat Container yang Berjalan:

sudo docker ps

Menghentikan Container:

sudo docker stop cowrie

Menghapus Container:

sudo docker rm cowrie

Menjalankan Kembali Container:

sudo docker start cowrie

Menjalankan dengan Log Real-time:

sudo docker start -a cowrie

Restart Container:

sudo docker restart cowrie

Konfigurasi Lanjutan (Opsional)

Custom Port Mapping:

Jika ingin menggunakan port yang berbeda:

sudo docker run -d \
  --name cowrie \
  -p 2222:2222 \
  cowrie/cowrie:latest

Volume Mapping untuk Konfigurasi Custom:

sudo docker run -d \
  --name cowrie \
  -p 22:2222 \
  -v /path/to/custom/cowrie.cfg:/cowrie/cowrie-git/etc/cowrie.cfg \
  cowrie/cowrie:latest

Monitoring dan Analisis

Cek Log Files dalam Container:

sudo docker exec cowrie tail -f /cowrie/cowrie-git/var/log/cowrie/cowrie.log

Download Log Files dari Container:

sudo docker cp cowrie:/cowrie/cowrie-git/var/log/cowrie/ ./cowrie-logs/

Monitoring Resource Usage:

sudo docker stats cowrie

Troubleshooting

Port 22 Already in Use:

Pastikan SSH service tidak menggunakan port 22:

sudo netstat -tulpn | grep :22

Container Tidak Berjalan:

Cek log untuk error details:

sudo docker logs cowrie

Permission Issues:

Pastikan user memiliki akses ke Docker:

sudo usermod -aG docker $USER

Keamanan dan Best Practices

  1. Isolasi Jaringan: Jalankan Cowrie di jaringan terisolasi
  2. Monitoring: Pantau log secara berkala untuk deteksi serangan
  3. Backup Konfigurasi: Simpan konfigurasi custom di luar container
  4. Update Regular: Update Docker image secara berkala
  5. Firewall: Batasi akses ke port Cowrie sesuai kebutuhan

Kesimpulan

Cowrie honeypot telah berhasil diinstal dan berjalan menggunakan Docker. Dengan setup ini, Anda dapat:

  • 🎣 Menangkap aktivitas malicious dan serangan brute force
  • 📊 Menganalisis pola serangan dan teknik yang digunakan
  • 🔍 Memantau log real-time untuk deteksi dini
  • 🐳 Mengelola dengan mudah menggunakan Docker container

Honeypot seperti Cowrie sangat berharga untuk penelitian keamanan dan memahami taktik yang digunakan oleh attacker di wild.
